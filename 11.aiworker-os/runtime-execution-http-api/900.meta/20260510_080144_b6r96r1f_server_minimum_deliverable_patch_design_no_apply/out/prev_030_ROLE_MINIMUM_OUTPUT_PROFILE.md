# ロール別 最低成果物プロファイル 正本案

## 1. 共通

全ロールで成果物本文は必須。

ただし、ロールごとに成果物の型が違う。

## 2. Worker

最低成果物。

- 成果物タイトル
- 概要
- 本文
- 確認ポイント
- 未解決事項
- 次工程

低性能Worker。

- 安定した標準成果物
- 一般的な内容
- 専門性は控えめ
- CX参照は浅い
- 独自提案は少ない

高性能Worker。

- 専門性が高い
- CX参照が深い
- 代替案やリスクを含む
- レビュー観点が多い

## 3. Leader

最低成果物。

- 中項目
- 作業単位候補
- Workerへの引き渡し内容
- 必要成果物
- 品質確認観点
- 未解決事項
- 次工程

低性能Leader。

- 安定した標準分解
- 粒度はやや粗い
- 依存関係の抽出は浅い
- Worker指示は標準的

高性能Leader。

- 分解粒度が適切
- 依存関係を整理
- Worker割当精度が高い
- 差し戻しリスクを下げる

## 4. Manager

最低成果物。

- 大項目
- 目的
- 優先度
- Leaderへ渡す観点
- 部門/役割候補
- 未解決事項
- 次工程

低性能Manager。

- 安定した粗い分割
- 方向性は一般的
- 部門間依存の把握は浅い
- 予測は控えめ

高性能Manager。

- 業務全体視野が広い
- 分割粒度が適切
- 優先度判断が強い
- リスク予測が深い
- 部門間依存を整理できる

## 5. President

最低成果物。

- 方針案
- 目的
- 優先順位
- 成功条件
- 制約条件
- Managerへ渡す方向性
- 未解決事項
- 次工程

低性能President。

- 安定した基本方針
- ビジョンは短期寄り
- 予測は控えめ
- 一般的な判断

高性能President。

- 長期ビジョン
- 事業判断
- 複数部門の統合
- 予測精度
- リスク許容度の調整
- 戦略的判断

## 6. 失敗時の共通成果物

どのロールでも通常成果物が作れない場合は、作業不能理由レポートを返す。

- なぜできないか
- 何が不足しているか
- どの権限が足りないか
- どの入力が必要か
- 次に何をすれば実行可能か
