# AIWorkerOS 安定成果物最低保証 正本案

## 1. 方針

低性能ロボットでも、成果物は安定したものを返す。

低性能とは、成果物が壊れる、本文がない、構成が崩れる、という意味ではない。

低性能とは、独自性、専門性、参照範囲、深掘り、予測精度、レビュー精度が控えめである、という意味で扱う。

## 2. 最低保証

全課金対象ロボットは、最低でもレビュー可能な成果物本文を返す。

必須項目は以下。

- deliverable_title
- deliverable_kind
- body_format
- body_markdown または body_text
- summary_text
- limitations_text
- unresolved_issues_text
- next_steps_text

## 3. 安定性の保証

低性能ロボットでも以下は満たす。

- 空成果物にしない
- summaryだけで終わらせない
- acceptedだけで終わらせない
- 作成条件だけを成果物本文にしない
- 章立てを崩さない
- 最低限の結論と次工程を出す
- 不足情報がある場合は不足情報レポートとして成果物化する

## 4. 性能差の出し方

性能差は成果物の安定性ではなく、以下で出す。

- CX22073JW参照権限
- CX参照深度
- CX参照範囲
- 専門性
- 独自提案力
- 予測精度
- レビュー精度
- リスク抽出精度
- 構成力
- 統合判断力

## 5. 低性能ロボットの成果物

低性能ロボットは、安定した標準成果物を返す。

特徴は以下。

- 一般的で分かりやすい
- 標準構成
- 専門的な踏み込みは少ない
- 独自提案は少ない
- CX参照は軽量または限定
- 予測やリスク評価は控えめ
- 不確実な点を明記する

## 6. 高性能ロボットの成果物

高性能ロボットは、安定した成果物に加えて以下を持つ。

- 深い専門性
- 広いCX参照
- 複数観点の統合
- 独自提案
- 代替案
- リスクと対策
- 次工程の具体化
- 精度の高いレビュー
- 将来予測や影響分析

## 7. 不可時の扱い

実行不能な場合でも、成果物本文を空にしない。

その場合は、作業不能理由レポートを成果物として返す。

含める内容。

- 結論
- 実行できない理由
- 不足情報
- 安全境界
- 権限境界
- 次に必要な入力
- 可能な代替案

## 8. 禁止

以下は禁止。

- 低性能だから成果物なし
- 低性能だから本文なし
- 低性能だから壊れた成果物
- 低性能だから作成条件だけ
- CX参照できない情報を参照したふりをする
- 専門性がないのに専門判断したふりをする
