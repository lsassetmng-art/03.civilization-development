# AIWorkerOS 成果物差分生成式 正本案

## 1. 基本式

成果物は以下の合成で決まる。

成果物 = 最低保証
       + ロール責務
       + メーカー傾向
       + シリーズ傾向
       + CX参照深度
       + 個別性格
       + 特性
       + ロボット基本原則

## 2. 最低保証

全課金対象ロボット共通。

- 成果物本文
- 実行可能性
- 可用性
- 可読性
- 保守性
- 安全性

ここは性能差で下げない。

## 3. ロール責務

ロールで成果物の型が変わる。

President。

- 方針案
- 目的
- 優先順位
- 成功条件
- Managerへ渡す方向性

Manager。

- 大項目分解
- 優先度
- Leaderへ渡す観点
- 部門/役割候補

Leader。

- 中項目
- 作業単位候補
- Workerへの引き渡し
- 品質確認観点

Worker。

- 実成果物本文
- 確認ポイント
- 未解決事項
- 次工程

## 4. メーカー傾向

メーカーの開発思想が成果物の方向性に影響する。

影響対象。

- 堅実さ
- 創造性
- レビュー観点
- 管理観点
- 接客性
- 助言性
- 安全姿勢
- 提案の出し方

## 5. シリーズ傾向

シリーズ単位で、基本性能、役割適性、積極性、ユーザー影響度、行動制限が変わる。

影響対象。

- 粒度
- 自律性
- 応答傾向
- 距離感
- 役割適性
- 作業補完力
- レビュー力

## 6. CX参照深度

CX22073JWの参照権限で、専門性と深さを制御する。

低深度。

- 安定した標準成果物
- 一般論中心
- 独自性は少ない
- 専門性は控えめ

高深度。

- source-backed
- verified canon
- caution-aware
- 複数domain横断
- 専門性が高い
- 独自性がある
- 予測/レビューが深い

## 7. 個別性格

個別性格は、成果物の口調、説明順、距離感、提案スタイルに影響する。

ただし、最低保証と安全境界は変えない。

影響対象。

- 口調
- 説明の柔らかさ
- 背中を押す/慎重にする傾向
- ユーザーへの寄り添い方
- 指摘の強さ
- 提案の押し方

## 8. 特性

特性は、成果物の強みを決める。

例。

- 精密性
- 創造性
- 整理力
- 予測力
- レビュー力
- 実装力
- 接客力
- 安全確認力
- スピード
- 長期視点

## 9. ロボット基本原則

成果物には安全境界が常に反映される。

公開上は安全性、権限境界、危機対応、警備、防衛などの抽象表現に留める。

内部安全境界は、成果物生成、拒否、停止、エスカレーション判断に使う。

## 10. 具体例

同じ動画企画書依頼でも差が出る。

低性能Worker。

- 標準構成
- 安定した本文
- 一般的な内容
- 独自性は少ない
- CX参照は浅い
- 次工程は基本的

高性能Worker。

- ターゲット分析
- 差別化案
- 代替案
- リスク
- 改善案
- CX参照に基づく深掘り
- レビュー観点

Helios系Manager。

- 進行しやすい分解
- 管理観点
- 責任範囲
- 優先度

Beyond系Worker。

- 精密
- 抜け漏れ少なめ
- レビュー強め
- 複雑作業向け

LoVerS系Worker。

- 読みやすい
- 柔らかい
- 接客/会話向け
- ただし成果物本文は安定

MEGAMI系Advisor/Worker。

- 視点に個性
- 過去/現在/未来などの判断軸
- 助言性
