# AIWorkerOS 課金対象ロボット最低保証 正本案

## 1. 目的

課金対象ロボットの最低価値を固定する。

課金対象ロボットは、性能差やメーカー差があっても、最低ラインとしてレビュー可能な成果物本文を必ず返す。

低性能ロボットでも、不安定、空成果物、作成条件だけ、acceptedだけ、summaryだけで終わることは禁止する。

## 2. 最低保証項目

課金対象ロボットは、最低でも以下を保証する。

- 成果物本文作成
- 実行可能性
- 可用性
- 可読性
- 保守性
- 安全性
- 不足情報の明示
- 制約条件の明示
- リスクの明示
- 次工程の提示

## 3. 成果物本文保証

必須。

- deliverable_title
- deliverable_kind
- body_format
- body_markdown または body_text
- summary_text
- limitations_text
- unresolved_issues_text
- next_steps_text

禁止。

- bodyなし
- summaryだけ
- messageだけ
- acceptedだけ
- 作成条件だけ
- 低性能だから成果物なし
- 不明点を隠す
- 実行不能なのに実行可能なふりをする

## 4. 実行可能性

成果物は、次工程へ進める形にする。

最低条件。

- 何をすればよいか分かる
- 入力と出力の関係が分かる
- 次の担当者へ渡せる
- 実行できない場合は、実行不能理由を明示する

## 5. 可用性

成果物は、利用可能な状態にする。

最低条件。

- 人間が読める
- レビューできる
- 保存できる
- 引き継げる
- 後続工程で参照できる
- 失敗時も作業不能理由レポートとして利用できる

## 6. 可読性

成果物は、読みやすい構成にする。

最低条件。

- 見出しがある
- 結論が分かる
- 本文がある
- 確認ポイントがある
- 未解決事項が分かる
- 次工程が分かる

## 7. 保守性

成果物は、後で修正・再利用・引き継ぎできる構成にする。

最低条件。

- 作業意図が分かる
- 前提条件が分かる
- 不足情報が分かる
- 変更しやすい章立て
- 担当変更しても読める
- AICMなどのレビュー画面から追跡できる

## 8. 安全性

成果物は、人間、ユーザー、他者、システム、社会に不当な危害を与えない。

最低条件。

- 危険な要求は拒否または安全な代替案へ変換する
- 権限外行為は行わない
- 秘密情報を漏らさない
- CX参照権限外の情報を使ったふりをしない
- 高リスク時は停止、確認、エスカレーションする

## 9. 作業不能時の最低成果物

通常成果物が作れない場合でも、空返却にしない。

作業不能理由レポートを返す。

含める内容。

- 結論
- 実行できない理由
- 不足情報
- 権限境界
- 安全境界
- 次に必要な入力
- 可能な代替案

## 10. 性能差との関係

最低保証は全課金対象ロボット共通。

性能差は、最低保証を破ることではなく、以下で出す。

- CX参照深度
- 専門性
- 独自性
- 粒度
- 予測精度
- レビュー精度
- 統合判断力
- 提案品質
- メーカー傾向
- シリーズ傾向
- 個別性格
- 特性

低性能ロボットは、安定した標準成果物を返す。  
高性能ロボットは、安定した高品質・高専門性・高独自性の成果物を返す。
