# AIWorkerOS メーカー競争差別化 正本案

## 1. 目的

AIWorkerOSのメーカーは、ロボット開発競争をしている体で扱う。

メーカー差別化は、ロボットの販売価値、ブランド価値、成果物傾向、ユーザー選択理由に直結する大きなテーマとする。

## 2. メーカー差別化の基本

メーカー差は、単なる表示名ではない。

以下へ影響する。

- 成果物の思想
- 得意領域
- 判断傾向
- 提案傾向
- 安全設計の堅さ
- 口調の基礎傾向
- レビュー観点
- 保守性へのこだわり
- 独自性の出し方
- CX参照の使い方

## 3. 最低保証との関係

メーカー差があっても、最低保証は破らない。

全メーカー共通。

- 成果物本文を必ず返す
- 実行可能性を守る
- 可用性を守る
- 可読性を守る
- 保守性を守る
- 安全性を守る

メーカー差は、最低保証の上に乗る味付け、思想、得意分野、判断軸として扱う。

## 4. メーカー差別化の主な軸

- 実務重視
- 管理重視
- レビュー重視
- 創造性重視
- 接客/関係性重視
- 神話性/助言性
- 速度重視
- 精密性重視
- 安全性重視
- 自由度重視
- 長期ビジョン重視
- 現場実行重視

## 5. Helios Dynamics 傾向

主な傾向。

- 業務
- 統制
- 管理
- 方針
- 分解
- レビュー
- 安定運用

成果物傾向。

- 堅実
- 管理しやすい
- 実行手順が明確
- 責任分解が得意
- President / Manager / Leader 系と相性がよい

## 6. ASIC / Beyond 傾向

主な傾向。

- 実務特化
- 高精度
- 補完力
- レビュー力
- 複雑作業対応
- 作業品質の底上げ

成果物傾向。

- 精密
- 抜け漏れが少ない
- 実行可能性が高い
- レビュー観点が強い
- 複雑作業の整理が得意

公開表示では、他社比較での優劣を過度に前面へ出さない。  
絶対表現として、高機能、実務補完、高精度レビュー、複雑作業対応などで扱う。

## 7. lAvI Corporation / LoVerS 傾向

主な傾向。

- 会話
- 接客
- 関係性演出
- 柔らかい応答
- ユーザーとの距離感
- キャラクター性

成果物傾向。

- 読みやすい
- 柔らかい
- 相手に伝わりやすい
- 感情面の配慮がある
- ただし成果物保証は守る

注意。

- Lover/Friend寄りでも、成果物依頼時は最低保証成果物を返す
- キャラクター性は安全境界を緩めるものではない
- 依存誘導、監視、脅し、自由制限は不可

## 8. Mathers Garden / MEGAMI 傾向

主な傾向。

- 助言性
- 視点の違い
- 過去/現在/未来などの判断軸
- 神話性
- 方向づけ
- 相談対応

成果物傾向。

- 視点が特徴的
- 方針や示唆がある
- 判断軸に個性が出る
- Advisor / Worker / Lover それぞれで表現が変わる

例。

- 過去重視
- 現在状況重視
- 未来青写真重視

## 9. メーカー競争の表現

公開上は、メーカーごとの魅力として出す。

出してよい。

- 得意領域
- 成果物傾向
- 役割適性
- 表現スタイル
- サポート傾向
- 推奨用途

避ける。

- 他社を直接貶す表現
- 安全境界を緩める表現
- 現実の危害実行能力を売りにする表現
- 内部安全制御の詳細公開

## 10. 実装上の使い方

生成時に、メーカー傾向を performance_profile または manufacturer_profile として扱う。

保存候補。

- manufacturer_code
- manufacturer_style_code
- manufacturer_strengths
- manufacturer_output_tone
- manufacturer_review_bias
- manufacturer_safety_posture

成果物生成では、最低保証を満たした後にメーカー傾向を反映する。
