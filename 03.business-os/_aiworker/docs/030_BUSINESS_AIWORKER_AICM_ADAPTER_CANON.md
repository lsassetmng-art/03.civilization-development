# BusinessOS AIWorker AICompanyManager Adapter Canon

## Purpose
This phase prepares the BusinessOS AIWorker module for AICompanyManager integration without directly modifying AICompanyManager UI files.

## Boundary
AICompanyManager must consume BusinessOS AIWorker through a connector/adapter layer.

## Canonical responsibilities

### AIWorkerOS
Owns:
- robot model canon
- series
- manufacturer
- personality
- safety boundary
- release state

### BusinessOS _aiworker
Owns:
- Business-side robot pool
- company entitlement
- role placement
- selector options
- AICompanyManager connector payload

### AICompanyManager
Consumes:
- selector options
- entitlement grant payload
- placement payload
- display label such as nickname@role

## Role selector rule
Role-specific selector must return only suitable robot models when recommended_role_codes exist.

Examples:
- President -> HD-R5
- Manager -> HD-R5 / HD-R4
- Leader -> HD-R4
- Worker -> HD-R3 / MEGAMI
- Helper -> HD-R1
- Friend -> HD-R1C
- Specialist -> MEGAMI

## UI integration rule
Do not free-text robot model codes in AICompanyManager.
User-facing UI must select from combo-box options generated by BusinessOS _aiworker.

## Display rule
Assigned robot display label:
internal_nickname@role_code

Example:
- Zeus@President
- Sato@Manager
- Yamada@Worker

## Quantity rule
- Pool and entitlement are quantity-based.
- Placement is one row per role assignment.
- Do not create one row per robot unit by default.
